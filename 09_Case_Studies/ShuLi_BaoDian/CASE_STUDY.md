# 案例研究：数理宝典

## 背景

数理宝典是一款面向江苏中考冲刺场景的离线数学公式学习 App。项目目标不是简单罗列公式，而是把公式查询、记忆复习、重点整理和策略闯关组织成一个可持续使用的学习工具，帮助初中生在复习阶段更快定位知识点，并通过轻量练习巩固解题思路。

项目采用零成本运行原则，不依赖付费后端、云数据库或云函数。核心公式库和闯关题库使用本地 JSON 管理，收藏、重点标记、复习进度和闯关记录通过本地缓存持久化。

## 核心产品问题

中考数学复习中的公式学习常见问题包括：

- 公式分散在教材、笔记和练习题中，查找效率低。
- 学生知道公式名称，但不一定知道适用条件和使用场景。
- 背诵和刷题割裂，缺少从公式到解题策略的连接。
- 复习进度、薄弱项和重点内容难以被持续记录。

数理宝典围绕这些问题设计为一个离线、轻量、可扩展的学习闭环。

## 产品方案

项目将学习路径拆成四个主要模块：

- 公式词典：按学段、模块、专题浏览公式，支持公式名称、关键词、别名和拼音首字母检索。
- 闪卡复习：通过翻卡、自评熟练度和最近练习记录，承接公式记忆需求。
- 闯关玩法：以“策略解题”为核心，把公式卡与题目场景连接起来，训练解题路径判断。
- 我的页面：沉淀收藏、重点标记、最近练习和薄弱项，形成个人复习视图。

这个结构让产品不是单点工具，而是从“查公式”延伸到“记公式、用公式、复盘公式”的闭环。

## 系统与落地

项目基于 `uni-app + Vue 3 + TypeScript + Pinia` 实现，兼顾 H5 和 App 构建链路。主要设计取舍包括：

- 使用本地数据文件维护公式库和闯关题库，降低运行成本和部署复杂度。
- 使用 Pinia 组织学习状态，通过本地缓存保存收藏、重点、复习和闯关进度。
- 通过统一的页面入口约定，让根目录页面和 `src/pages` 包装入口保持一致，降低后续扩展风险。
- 使用自绘底部导航和统一二级页返回逻辑，保证 App 端导航体验一致。
- 保留 H5、App 和本地 APK 构建链路，便于不同阶段演示和验证。

## 我的贡献

- 产品定义：明确“离线数学公式学习 App”的范围，控制成本边界，聚焦中考冲刺高频场景。
- 信息架构：设计首页、公式词典、闪卡、闯关、搜索和我的页面之间的学习路径。
- 交互与体验：梳理底部导航、二级页返回、公式卡展示、长公式阅读、收藏和重点标记等关键体验。
- 技术落地：组织 `uni-app` 项目结构、本地数据、状态持久化、页面入口和构建脚本。
- 交付沉淀：补充页面接入约定、手工冒烟清单和项目状态说明，方便后续维护和接手。

## 当前交付状态

当前项目已形成可运行版本，已覆盖公式浏览、全局搜索、闪卡复习、闯关玩法、收藏重点、个人复习视图和本地持久化。构建链路已验证 H5、App 和本地 APK 方向，后续可继续扩充公式库、题库和真机回归测试。

## 作品集价值

数理宝典展示的是一个从学习场景出发的轻量产品落地能力：在没有后端预算的约束下，仍然能够完成清晰的信息架构、可维护的本地数据模型、可验证的跨端构建链路和可持续迭代的功能闭环。

它补充了作品集中偏业务平台、运营工具和数据处理之外的另一个维度：面向具体学习场景，把产品规划、用户体验和前端工程落地结合起来。
