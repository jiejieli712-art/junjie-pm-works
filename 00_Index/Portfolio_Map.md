# 作品集地图

这份作品集按产品经理的工作路径组织：先定义问题，再理解用户与市场，然后设计体验、梳理系统、沉淀数据与交付证据。

## 阅读顺序

1. `09_Case_Studies/`：先看核心案例，理解项目为什么存在。
2. `01_Product_Strategy/`：查看 PRD、产品策略、商业判断和路线图材料。
3. `03_User_Experience/`：查看信息架构、用户流程、原型和高保真稿。
4. `04_System_Architecture/`：查看产品结构、后台逻辑和接口协作说明。
5. `06_Data_and_Operations/`：查看商家、菜品、运营数据处理材料。
6. `08_Scripts_and_Tools/`：查看辅助落地的脚本和自动化工具。
