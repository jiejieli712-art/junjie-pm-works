## 项目 1：干饭吧 / Bitego 小程序与后台协同项目

### 项目名
干饭吧：趣味选餐小程序与后台协同管理项目

### 项目类型
产品需求梳理 + 小程序页面落地 + 后台业务配置协同

### 你的角色
产品+技术复合型核心推动者，负责需求梳理、原型/流程沉淀、前端页面实现，并参与后台业务模块协同落地。

### 业务目标
面向外卖与集体用餐场景，解决“选择困难、选餐效率低、体验单调”的问题，通过趣味玩法、个性化推荐和批量预订，提升用户选餐体验与平台粘性。

### 你做的关键动作
- 基于 PRD 梳理出产品目标、用户角色和功能边界，围绕趣味玩法选餐、个性化推荐、多日预订等核心能力推进方案落地。
- 结合高保真原型、流程图和页面草图，推动首页、选餐页、订阅预订页、个人中心等核心页面实现。
- 参与小程序前端开发，落地 `home / select / subscribe / mine` 等页面及转盘结果、推荐预订、地址弹窗等业务组件。
- 结合后台项目结构，协同业务配置、用户偏好、菜品、反馈、推广等模块的管理能力落地。
- 整理路演材料、API 文档和补充方案文档，为产品评审、协同开发和后续展示提供统一材料。

### 可证明的交付物

## 项目 3：店铺 / 菜品数据处理与商家匹配项目

### 项目名
餐饮商家与菜品数据采集、清洗及结构化处理项目

### 项目类型
数据采集 + 数据清洗 + 结构化交付

### 你的角色
数据处理方向的主导执行者，负责数据抓取脚本、商家匹配、结果整理和结构化输出。

### 业务目标
围绕餐饮业务选品、商家匹配和区域数据整理需求，采集并整理多城市商家数据和菜品基础数据，为后续业务分析、选餐推荐和运营决策提供底层数据支持。

### 你做的关键动作
- 基于高德地图周边搜索接口编写网格化检索脚本，按城市边界、搜索半径和分页策略抓取餐饮 POI 数据。
- 生成并维护 `poi_list.csv`、按区县拆分的 JSON/CSV、匹配结果与汇总文件，支撑后续清洗和分析。
- 处理徐州及 Top20 城市商家数据，完成二次匹配、未匹配结果识别和区县维度结构化输出。
- 结合菜品来源数据、脑图和字段表，沉淀菜品库整理思路，为推荐和运营规则提供基础数据材料。
- 通过脚本化方式降低重复整理成本，提升多区域、多批次数据处理的一致性。

### 可证明的交付物
- 代码：`AmapGridSearch.java`、`amap-grid-search` 工程。
- 数据结果：`poi_list.csv`、`all_matched.csv/json`、`summary.csv/json`、区县级输出目录。
- 中间过程：`keyword_cache`、`keyword_output`、`keyword_progress`。
- 桌面侧结果：徐州匹配结果、Top20 城市分区商家 CSV/JSON、菜品数据来源与相关脑图。

### 技术栈
- Java、HttpClient、Hutool JSON
- CSV、JSON、Excel
- 高德地图 API
- 数据清洗、结构化输出、目录化交付

### 30字标题版
主导餐饮商家数据采集与匹配整理，输出可复用的结构化结果集。

### 80-120字项目简介版
该项目主要服务于餐饮业务的数据准备需求，我基于高德地图接口和脚本化处理方式，完成商家 POI 数据采集、区县维度拆分、结果匹配与汇总输出，并结合菜品来源与字段资料沉淀数据底稿，为后续运营分析、菜品整理和推荐逻辑提供支撑。

### 简历 Bullet 版

Java、HttpClient、Hutool、Amap API、CSV、JSON、Excel、数据清洗、结构化输出
