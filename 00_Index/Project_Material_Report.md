## 二、核心项目资产

### 1. 干饭吧 / Bitego

这是目前材料中最完整、最适合写进简历和作品集的项目线。资料覆盖 PRD、方案文档、流程图、高保真原型、小程序源码、后台管理系统源码、商业计划书和路演材料。

可体现的能力：

- 能从用餐场景痛点出发，梳理趣味选餐、个性化推荐、多日预订等产品功能。
- 能将 PRD、流程图和高保真原型转化为小程序页面与组件。
- 能理解并协同后台模块，包括用户偏好、菜品管理、反馈、推广配置等。
- 具备从产品设计、页面实现到项目展示材料整理的完整交付意识。

### 2. 元来短剧 / 短剧平台

该项目线覆盖短剧平台二开需求文档、短剧源码、用户端工程、后台管理端工程、服务端结构说明、隐私材料和申报材料。项目形态更接近内容分发与运营管理平台。

可体现的能力：

- 能理解短剧内容平台的用户端、后台端、服务端协同关系。
- 能梳理用户观看、关注、历史、收藏、充值、登录注册等用户链路。
- 能理解后台剧目、剧集、分类、评论、用户、权限等运营管理模块。
- 能参与二开需求理解、页面结构梳理和项目交付材料沉淀。

### 3. 店铺 / 菜品数据处理

该项目线包含高德地图商家采集脚本、徐州及多城市商家 CSV/JSON、匹配结果、区县维度输出、菜品数据来源和算法/数据整理材料。

可体现的能力：

- 能基于高德地图接口进行餐饮商家数据采集。
- 能完成商家数据清洗、匹配、区县拆分和结构化输出。
- 能处理 CSV、JSON、Excel 等多格式数据。
- 能为餐饮运营、菜品整理和后续推荐逻辑提供数据底稿。

## 四、简历定位结论

基于现有材料，最合适的简历方向是“产品 + 技术复合型”，尤其适合投递：

- 产品助理实习
- 小程序产品实习
- 前端偏业务落地实习
- 产品运营支持
- 初级项目执行 / 交付支持类岗位

推荐简历主线：

1. 用“干饭吧 / Bitego”展示从需求、原型、页面实现到后台协同的完整闭环。
2. 用“元来短剧 / 短剧平台”展示跨端平台型业务理解和二开落地能力。
3. 用“店铺 / 菜品数据处理”展示数据采集、清洗和业务支持能力。

## 五、风险与注意事项

- 不建议直接把完整压缩包发给外部 AI 或招聘方，里面可能包含账号、接口、缓存、原始数据和内部材料。
- 简历中不要写无法证明的硬指标，例如 DAU、转化率、营收、准确率等。
- 后端和全栈能力建议写成“理解并协同”或“参与落地”，不要夸大为独立负责完整后端架构。
- 数据处理项目适合写成数据采集、清洗和结构化输出，不建议包装成完整推荐系统或算法平台。
