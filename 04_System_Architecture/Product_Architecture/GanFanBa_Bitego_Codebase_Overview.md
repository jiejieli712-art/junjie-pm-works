# 干饭吧 / Bitego 代码证据概览

本文件用于说明干饭吧项目对应的代码材料如何支撑作品集叙事。仓库中不上传完整工程源码，只保留能证明技术落地路径的轻量证据和结构摘要。

## 代码材料来源

- 后端工程：`bitego-master`
- 后台前端工程：`bitego-web-ui`
- 本仓库收录证据：后台前端 `package.json`、代码结构摘要、接口/架构说明

## 系统组成

干饭吧代码材料体现的是一套“小程序业务 + 后台管理 + 数据处理”的组合系统。

核心模块包括：

- 后台管理：用户、角色、菜单、系统配置、日志、任务调度等基础管理能力。
- 业务模块：菜品、标签、客户、抽奖/玩法、推广等与干饭吧业务相关的管理接口。
- 数据支撑：菜品字段结构、偏好标签、商家匹配结果和区域采集摘要。
- 前端后台：基于 Vue 3、Vite、Element Plus、Pinia、Vue Router 和 ECharts 的管理端页面。

## 技术栈摘要

后端侧：

- Java 8
- Spring Boot 2.7
- Spring Security
- MyBatis Plus
- MySQL
- Druid
- PageHelper
- Swagger
- JWT
- OSS / 图片处理 / Excel 处理

前端侧：

- Vue 3
- Vite
- Element Plus
- Pinia
- Vue Router
- Axios
- ECharts
- Quill 富文本

## 可支撑的作品集表达

- 能把 PRD 中的点餐、偏好、玩法、后台管理需求拆成前后台模块。
- 能理解后台管理系统的通用能力和业务模块之间的边界。
- 能围绕菜品、标签、客户、推广等业务对象组织管理端能力。
- 能从产品文档、接口说明、数据结构和代码依赖中建立交付证据链。

## 不上传完整源码的原因

- 完整工程包含大量框架模板、依赖、构建产物和内部配置，不适合作为公开作品集主体。
- 作品集更需要展示产品判断、系统理解和交付证据，而不是堆放完整源码。
- 如需展示实现能力，优先展示关键模块说明、接口关系、数据结构和少量脱敏脚本。
